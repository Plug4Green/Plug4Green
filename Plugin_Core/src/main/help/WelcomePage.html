How do I launch the FIT4Green plug-in in DEMO mode?

You can take a first taste of how the plug-in operates by launching it in DEMO mode. To do so, simply follow the steps below:

  1. Open the FIT4Green Management UI on your browser: 
    http://localhost:8080/F4gGui/
  2. Start the plug-in by pressing "Start"

How do I install the FIT4Green plug-in?

You can follow the simple steps below:

  1. Install the contents of this CD on a new (virtual) machine. Please see here for details on how to configure the network of the host machine.
  2. Boot the new (virtual) machine and perform the next steps from within it
  3. Stop the Tomcat server; on a terminal you can run: 
    /home/fit4green/Fit4Green/tomcat6/bin/shutdown.sh
  4. Edit the FIT4Green configuration files; for details see: 
    /home/fit4green/Fit4Green/tomcat6/common/classes/config/*.properties
  5. Start the Tomcat server; on a terminal you can run: 
    /home/fit4green/Fit4Green/tomcat6/bin/startup.sh
  6. Open the FIT4Green Management UI on your browser: 
    http://localhost:8080/F4gGui/
  7. If applicable, start the plug-in by pressing "Start"

How do I use the built-in development environment?

You can launch eclipse: 

/home/fit4green/Fit4Green/eclipse
The workspace includes the source code of the main components of the plug-in. It is in effect the snapshot of the development environment that was used to built this live CD.

Please note that changes cannot be committed to the code repository, since it is just an export and not a checkout.

Where can I find the FIT4Green log files?

The default location for the FIT4Green log file is: 

/home/fit4green/Fit4Green.log
You can change the location of this file along with other logging possibilities by updating the file: 
/home/fit4green/Fit4Green/tomcat6/common/classes/config/log4j.properties

top

When opening the configuration files on the Configuration tab, I get a message if I want to save changes even though I made none. What is this all about?

This is caused by the browser on Linux altering the line-end characters from the files which are most likely created on Windows. It can be solved by opening and saving each of the files once from within Linux.

How do I take a look at the databases where historical data are stored?

The plug-in manages programmatically the couchDB databases; so there is no need to perform any administration tasks per se. Nevertheless, in case you wish to browse through the databases, CouchDB comes with a built-in interface called Futon. To load Futon in your browser, visit: 

http://localhost:5984/_utils/index.html

How do I configure the plug-in for use with one of the already developed communication modules?

The FIT4Green live CD includes the necessary libraries for running the plug-in at the three FIT4Green pilot sites: a service/enterprise portal at Eni S.p.A. (Italy), supercomputing grid at Juelich Supercomputing Centre (Germany) and cloud computing data centres at HP Italy Innovation Centre (Italy).

You can configure the plug-in for use with one of the already developed communication modules by editing the FIT4Green configuration file: 

/home/fit4green/Fit4Green/tomcat6/common/classes/config/f4gconfig.properties

For more information on the communication module related to the service/enterprise portal at Eni, see here.

For more information on the communication module related to the supercomputing grid at Juelich, see here.
